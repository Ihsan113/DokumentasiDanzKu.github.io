H2H Service API Documentation

A powerful API for managing prepaid services, deposits, and transactions.

Last updated: September 23, 2025

Overview

The H2H Service API allows you to integrate prepaid services, deposit methods, and transaction management into your application. Powered by Atlantic Pedia, it provides real-time access to pricing, deposit options, and transaction processing with user-specific fee adjustments.

Base URL: https://your-domain.com/api
Authentication: Requires an API key in the request body.
Support: Email us at support@yourdomain.com.

---

Quick Start

To get started, you need:

  • A valid API key from your account dashboard.
  • Tools like Postman, curl, or a custom HTTP client.
  • Node.js environment (optional for testing).

Include your API key in the request body:

{
    "api_key": "your_api_key_here"
}

Endpoints

1. Get Price List

Retrieve a list of services with adjusted prices.

Method: POST | URL: /layanan/price_list

Show Details

Parameters (Body)

Parameter Type Required Description Example
api_key string Yes Your API key abc123def456
type string Yes Service category prabayar
code string No Product code PULSA10K

Request Example

curl -X POST https://your-domain.com/api/layanan/price_list \
 -H "Content-Type: application/json" \
 -d '{
  "api_key": "your_api_key_here",
  "type": "prabayar",

Webhooks

Layanan webhook kami memungkinkan Anda menerima notifikasi secara real-time untuk status **transaksi** dan **deposit** Anda.

URL yang harus Anda sediakan: https://your-domain.com/webhook

Metode: POST
Content-Type: application/json

1. Notifikasi Transaksi

Anda akan menerima notifikasi ini ketika status transaksi berubah.

Show Details

Contoh Payload (berhasil)

{
  "event": "transaksi",
  "data": {
    "reff_id": "TRANS-20250919-XYZ",
    "layanan": "Pulsa Telkomsel 10K",
    "target": "081234567890",
    "status": "success",
    "sn": "A1B2C3D4E5F6G7H8I9J0"
  }
}

Contoh Payload (gagal)

{
  "event": "transaksi",
  "data": {
    "reff_id": "TRANS-20250919-XYZ",
    "layanan": "Pulsa Telkomsel 10K",
    "target": "081234567890",
    "status": "failed",
    "sn": "N/A"
  }
}

Penjelasan Payload

Kunci Tipe Data Deskripsi
event string Selalu `transaksi` untuk notifikasi ini.
data object Objek yang berisi detail transaksi.
data.reff_id string ID referensi unik untuk transaksi.
data.layanan string Nama produk atau layanan yang dibeli.
data.target string Nomor tujuan atau akun yang diisi.
data.status string Status transaksi. Nilai yang mungkin: `success` (berhasil) atau `failed` (gagal).
data.sn string Serial number atau kode unik transaksi. `N/A` jika gagal.

2. Notifikasi Deposit

Anda akan menerima notifikasi ini ketika status deposit berubah.

Show Details

Contoh Payload (berhasil)

{
  "event": "deposit",
  "data": {
    "reff_id": "DEPO-20250919-ABC",
    "metode": "QRIS",
    "status": "success"
  }
}

Contoh Payload (dibatalkan)

{
  "event": "deposit",
  "data": {
    "reff_id": "DEPO-20250919-ABC",
    "metode": "QRIS",
    "status": "canceled"
  }
}

Penjelasan Payload

Kunci Tipe Data Deskripsi
event string Selalu `deposit` untuk notifikasi ini.
data object Objek yang berisi detail deposit.
data.reff_id string ID referensi unik untuk deposit.
data.metode string Metode pembayaran yang digunakan, contoh: `QRIS`, `VA BNI`.
data.status string Status deposit. Nilai yang mungkin: `success` atau `processing` (keduanya menandakan deposit berhasil masuk), atau `canceled` (dibatalkan).

Error Handling

Errors follow this structure:

{
  "status": false,
  "message": "Descriptive error message",
  "error": { "details": "Optional error object" }
}

Common codes:

  • 400: Bad Request (e.g., missing parameters)
  • 401: Unauthorized (invalid API key)
  • 403: Forbidden (expired API key)
  • 404: Not Found (e.g., invalid product code)
  • 500: Server Error
  • 503: Service Unavailable (external API down)
---

Best Practices

  • Validate Inputs: Check parameters client-side.
  • Handle Timeouts: Use 10-15s timeouts.
  • Monitor Balance: Poll dashboard for updates.
  • Testing: Use sandbox mode (contact support).
  • Security: Never expose API keys in client code.
